# Урок 1: Изоляция транзакций и расширения PostgreSQL

## Изоляция транзакций

### Уровни изоляции в PostgreSQL

PostgreSQL реализует три уровня изоляции транзакций (хотя стандарт SQL определяет четыре):

1. **Read Uncommitted** - в PostgreSQL работает как Read Committed
2. **Read Committed** - уровень по умолчанию
3. **Repeatable Read**
4. **Serializable** - самый строгий уровень

### Особые явления при параллельных транзакциях

Стандарт SQL определяет следующие явления, которые недопустимы на разных уровнях:

- **Dirty Read** - чтение данных незавершённой транзакции
- **Non-repeatable Read** - повторное чтение показывает изменённые данные
- **Phantom Read** - повторный запрос возвращает разный набор строк
- **Serialization Anomaly** - несогласованные результаты при параллельном выполнении

### Read Committed (по умолчанию)

- SELECT видит данные, зафиксированные до начала запроса
- Последовательные SELECT могут видеть разные данные в одной транзакции
- UPDATE/DELETE находят строки, зафиксированные на момент начала команды

### Repeatable Read

- Все SELECT в транзакции видят снимок данных на момент начала транзакции
- Предотвращает все три вида аномалий чтения
- При попытке изменить строку, изменённую другой транзакцией, происходит откат с ошибкой сериализации

### Serializable

- Самый строгий уровень изоляции
- Моделирует последовательное выполнение транзакций
- Дополнительная проверка на аномалии сериализации
- При обнаружении конфликтов транзакция откатывается

## Расширения PostgreSQL

### Что такое расширения

Расширения - это пакеты SQL объектов, которые:
- Добавляют новую функциональность в PostgreSQL
- Упрощают управление связанными объектами
- Поддерживаются pg_dump (сохраняются как CREATE EXTENSION)
- Можно обновлять с помощью ALTER EXTENSION UPDATE
- Нельзя удалить отдельные объекты расширения (только всё расширение)

### Популярные расширения

#### 1. PostGIS
- Пространственные данные: geometry, geography, raster
- Геопространственные запросы и функции
- Применение: IoT, трекинг, геолокационные сервисы

#### 2. pg_stat_statements
- Статистика выполнения запросов
- Анализ производительности и медленных запросов
- Включает shared_blks_hit/shared_blks_read для анализа I/O

#### 3. pgcrypto
- Криптографические функции
- Хеширование, шифрование, дешифрование
- Алгоритмы: DES, 3DES, AES, Blowfish

#### 4. pg_partman
- Автоматическое управление партиционированием
- Поддержка временных и ID-based партиций
- Автоматическое создание/удаление партиций

#### 5. postgres_fdw
- Foreign Data Wrapper для подключения к другим PostgreSQL
- Федеративные запросы между базами данных

#### 6. pgvector
- Векторные операции и поиск по сходству
- Применение: AI, рекомендательные системы, NLP
- Операторы расстояния: <->

#### 7. hstore
- Хранилище ключ-значений в PostgreSQL
- Полуструктурированные данные
- GIN индексы для быстрого поиска

#### 8. pgpcre
- Perl Compatible Regular Expressions
- Расширенные возможности регулярных выражений
- Применение: анализ текста, валидация форматов

### Установка расширений

```sql
-- Просмотр доступных расширений
SELECT * FROM pg_available_extensions;

-- Установка расширения
CREATE EXTENSION имя_расширения;

-- Просмотр установленных расширений
SELECT * FROM pg_extension;

-- Удаление расширения
DROP EXTENSION имя_расширения;
```

### TimescaleDB

TimescaleDB - это расширение PostgreSQL для временных рядов, которое:
- Делит данные на chunks по временным интервалам
- Сжимает данные для оптимизации
- Поддерживает параллельную обработку
- Включает функции для анализа временных рядов
